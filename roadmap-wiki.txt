# DWeb LivelyKernel  
  
## Introduction  
  
 
The project **Lively Kernel** is a **web-native programming environment and operating system-like runtime that runs inside a browser**. It was originally developed at **Sun Labs** by people connected to the Smalltalk/Self research community. and later --**SAP** and continued development at the **6-2018 [HPI Software Architecture Group](http://www.hpi.uni-potsdam.de/swa) Hasso Platter Instute**--later add **YCR** ..... ???
  **DWeb LivelyKernel** is a fork of LivelyKernel enhanced to support **decentralized web protocols**, cryptographic identity, and content-addressed worlds.  
  
### Design Goals  
- WebAuthn based login
- Decentralized verifiable Credentials for user profiles and permissioning  
- IPFS/IPNS world storage with IPLD DAG JSON snapshots  
- Offline Local Network First, multi-user collaboration support
- Polished and refreshed Lively morphic UI and runtime (halos, scripts, morphs etc.)

### 3. Decentralized Cryptographic Identity  
- DID:PKH generation  (`did:pkh:eip155:1:0x...`)
- Verifiable Credentials (VCs) issued for profile, permissions  (`did:web`) etc.
- Every **user** is represented by a **DID**:
- Mapping .keo domain names to user DID document
- did:web:alice.example.computer -> resolves to: ipns://did:pkh:eip155:1:0xAlice/world
-    **IPNS** provides a mutable pointer to the latest world CID.
did:web:lively.computer  -> resolves to: ipns://<CID pointing to world DAG>

## Key Notes

-   Morphic object references are **immutable via CID**, allowing reproducible snapshots.
    
-   Users own their **DIDs**, which are directly mapped to **IPNS addresses**.
    
-   Updating a world creates **new CIDs**, while IPNS maintains a mutable pointer to the latest state.
    
-   All serialization is **DAG-JSON compliant**, compatible with IPFS/Helia and future CRDT integrations.
